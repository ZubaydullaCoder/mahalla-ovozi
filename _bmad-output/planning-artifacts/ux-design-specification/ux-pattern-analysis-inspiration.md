# UX Pattern Analysis & Inspiration

## Adopt

- **Telegram:** readable mixed-script evidence metadata and familiar source
  navigation.
- **Linear:** overlay drawer, stable selection, and skeleton replacement.
- **Kanban boards:** simultaneous multi-lane scanning with independent scroll.

## Adapt

- Cards represent canonical topics, not tasks or chat messages.
- One topic may render in several service lanes, but every copy opens the same
  topic and shows all category chips.
- The evidence drawer is chronological but not a conversational reply UI.
- Exact Telegram links are deliberate external verification actions.

## Avoid

- chat bubbles and alternating message layout;
- case status, assignee, severity, or resolution controls;
- charts that replace the evidence-first topic view;
- AI summaries without original evidence access;
- color-only category meaning;
- Hokim-lane styling that implies verified urgency;
- action menus on evidence rows.
